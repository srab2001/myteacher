## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | /api/plans/:planId/versions | List all versions for a plan |
| GET | /api/plans/:planId/versions/:versionId | Get specific version |
| GET | /api/plans/:planId/versions/current | Get current version |
| POST | /api/plans/:planId/versions | Create new version |
| PATCH | /api/plans/:planId/versions/:versionId | Update version metadata |
| GET | /api/plans/:planId/versions/:versionId/pdf | Download PDF |
| GET | /api/plans/:planId/versions/compare | Compare two versions |
| POST | /api/plans/:planId/versions/:versionId/finalize | Finalize version |
| GET | /api/plans/:planId/versions/:versionId/snapshots | Get section snapshots |

## Business Rules

1. Only one version can have status CURRENT per plan
2. FINALIZED versions cannot be edited - must create new version
3. PDF is generated automatically when version is finalized
4. Previous CURRENT version becomes SUPERSEDED when new version is finalized
5. Version numbers auto-increment and cannot be reused
6. Comparison only available between versions of same plan
7. All version changes logged to Decision Ledger
8. Signatures are required on FINALIZED versions
